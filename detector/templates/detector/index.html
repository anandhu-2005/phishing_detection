CLICK CARD FOR DETAILS

Our classifier extracts 12 structural metrics from the URL string. Click any card to expand a comprehensive explanation of why that metric triggers security alerts.

1. URL LENGTH SAFE
- expand_more
Suspicious domains are often unusually long (over 70 characters) due to embedding subdomain parameters, sub-pages, or spoofed brands to deceive visual inspections.
2. HAS IP ADDRESS SAFE
- expand_more
Legitimate sites rarely use raw IP addresses (e.g., 192.168.1.1) as the hostname. Using raw IPs is a typical attacker tactic to avoid registering standard DNS records.
3. HAS @ SYMBOL SAFE
- expand_more
The @ symbol forces browsers to ignore everything before it and route the user to the destination after it — a highly characteristic phishing trick.
4. DOUBLE SLASH POSITION SAFE
- expand_more
A double slash // appearing late in the URL path (after position 7) suggests structural manipulation — typically used to mask an open redirect destination.
5. SUBDOMAINS COUNT SAFE
- expand_more
Phishing sites stack subdomains (e.g., paypal.com.update.secure.net) to create a false sense of authority. More than 2 subdomains raises diagnostic flags.
6. HTTPS SCHEME SAFE
- expand_more
Lack of HTTPS exposes data to eavesdropping. Genuine login systems exclusively use HTTPS. Absence of TLS certificates is an extreme threat multiplier.
7. REG. DOMAIN LENGTH SAFE
- expand_more
The length of the main registered domain name. Phishers often register overly long or random domains to find unregistered variations resembling target brands.
8. PATH LENGTH SAFE
- expand_more
The length of the subdirectory path. Phishing links tend to use deep directories to push the real domain host away from the visible address bar.
9. HYPHEN IN DOMAIN SAFE
- expand_more
Hyphens (e.g., security-paypal.com) are rarely used in legitimate brand domains. Attackers insert hyphens to bypass word checks and mimic official titles.
10. URL DIGIT RATIO SAFE
- expand_more
Percentage of numeric characters in the URL. High digit concentration indicates randomized algorithms generating subpages typical of automated scam scripts.
11. SPECIAL CHARS COUNT SAFE
- expand_more
Dynamic query chars (?, =, &) and encoding chars (%) are heavily loaded in malicious URLs to forward stolen data while bypassing basic static filters.
12. TLD IN PATH STRING SAFE
- expand_more
Extensions like .com or .org appearing inside the directory path (e.g., domain.org/login.com) rather than the real domain segment indicate redirect spoofing.

Heuristics Extraction Documentation

info URL heuristics represent key structural behaviors extracted statically from the URL string. These parameters do not require a live HTTP connection, making our verification process completely isolated and safe for the scanning host.

HEURISTIC RULE THREAT RATIONALE STANDARD THRESHOLD
URL Length Attackers embed subpages, dynamic parameters, and spoofed keywords. Safe under 70 chars. Critical above.
Has IP Address Used to skip standard DNS registrations and hide hosting servers. Strictly binary (0 or 1). Any IP is high-risk.
Has @ Symbol Credentials routing redirection trick to spoof login domains. Critical if present.
Double Slash // Masks external open redirects, hiding true target destinations. Suspicious if position > 7.
Subdomains Count Simulates trusted host hierarchies (e.g., login.brand.secure.net). Standard: ≤ 2. Flagged if ≥ 3.
HTTPS Scheme Absence of secure TLS certs signals low-trust environments. High-risk if HTTP is detected.
Hyphen in Domain Legitimate institutions avoid hyphens. Spoofer domains use them extensively. High correlation if hyphen exists.
Digit Ratio Automated domain generation algorithms load URLs with numeric hashes. Flagged if ratio exceeds 12% of total length.
Special Chars High count of ?, =, &, % signals data-exfiltration query strings. Flagged if count exceeds 3.
TLD in Path Extensions (.com, .net) inside the path suggest redirect spoofing. Critical if TLD found in directory path.
